
# Система разграничения доступа (RBAC)

В приложении реализована система управления доступом по модели **Role-Based Access Control (RBAC)**.

## Общая схема

Доступ к ресурсам определяется через роли и права:
Пользователь → Роль → Права (permissions) → Доступ к endpoint

## Структура базы данных

Используются таблицы:

- **users** — пользователи  
- **roles** — роли (`user`, `admin`)  
- **permissions** — права (`read_profile`, `edit_profile`, `delete_user`)  
- **user_roles** — связь пользователей и ролей  
- **role_permissions** — связь ролей и прав  

---

## Реализация в коде

### Аутентификация (`encript.py`)

- `create_access_token(user_id)` — создаёт JWT  
- `decode_token(token)` — декодирует JWT  
- `get_current_user(...)` — получает пользователя из токена  
  - если токен отсутствует или неверный → **401 Unauthorized**


### Авторизация (`dependency.py`)

- `require_permission(permission_name)`

Проверяет наличие права у пользователя:

1. Получает пользователя через `get_current_user`
2. Проверяет его роли и права в БД
3. Если право есть → доступ разрешён  
4. Если нет → **403 Forbidden**

### Назначение ролей
При регистрации (`auth.py`)

Новому пользователю автоматически назначается роль user.
